# Vision

`<PROJECT_NAME>` should replace this file with a concrete statement of purpose.

As a starting point, every Codelab open-source project should answer:

- What problem is the project solving?
- Who is it for?
- Why should this exist as open source?
- What is intentionally out of scope right now?

Keep this document short and specific. If the project cannot explain its scope clearly, the repository will accumulate ambiguous issues and weak pull requests.
